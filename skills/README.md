# Skills

This directory is optional.

Use it when the target environment supports reusable procedural skills or workflows.

What lives here:

- `repo-orchestrator/`
  - routing workflow for truth, status, plans, research, decisions, worklogs, inbox intake, and deeper `mydocs/` notes
- `upstream-intake/`
  - companion workflow for the canonical upstream-review module

Keep skills procedural.
Do not duplicate the canonical rules from `repo-operating-model.md` inside them.
